# Seeder Pay 核心業務場景與功能驗證清單 (FUNCTIONAL_CHECKLIST.md)

本文件定義了 Seeder Pay 系統的關鍵業務路徑及其驗證 Checkpoints，用於確保全棧功能打通後的穩定性。

---

## 1. 核心業務鏈路 (Core Business Flows)

### 1.1 產品與定價管理 (Product & Pricing)
*   **路徑**：`Dashboard -> 產品/方案`
*   **業務鏈路**：
    1.  用戶點擊「新增產品」並填寫基本信息。
    2.  用戶為產品設置「價格」及「計費週期」（月繳/年繳/一次性）。
    3.  後端 `CreateProduct` -> `CreatePrice` 接口異步完成數據持久化（SQLite）。
    4.  前端列表實時刷新顯示新產品。

### 1.2 顧客管理 (Customer Lifecycle)
*   **路徑**：`Dashboard -> 顧客`
*   **業務鏈路**：
    1.  用戶手動新增顧客（Email/名稱）。
    2.  系統在數據庫中生成唯一的顧客 ID (`cus_...`)。
    3.  可在列表查看顧客的註冊日期與訂閱狀態。

### 1.3 訂閱與發票自動化 (Subscription & Invoicing)
*   **路徑**：`Dashboard -> 訂閱` & `Dashboard -> 訂單`
*   **業務鏈路**：
    1.  （當前模擬）訂閱創建後，系統自動生成 Initial Invoice（發票）。
    2.  訂閱狀態從 `active` 開始追蹤周期。
    3.  用戶可點擊「取消訂閱」，狀態變更為 `canceled` 或預約在周期結束時取消。

### 1.4 數據統計與概覽 (Stats & Overview)
*   **路徑**：`Dashboard -> 總覽`
*   **業務鏈路**：
    1.  前端進入頁面時調用 `/api/v1/stats/overview`。
    2.  後端通過 SQL 聚合計算：`SUM(amount_paid)` (總營收), `COUNT(active_subscriptions)` (活躍數), `SUM(monthly_unit_amounts)` (MRR)。
    3.  Dashboard 數字動態更新。

## 2. 功能驗證 Checkpoints

### ✅ 產品管理 Checkpoints
- [ ] **創建校驗**：提交空名稱應報錯；提交正整數金額應成功。
- [ ] **列表展示**：新產品應立即出現在表格首行。
- [ ] **編輯功能**：修改產品描述後，列表應顯示更新後的文本。
- [ ] **停用功能**：點擊「停用」後，產品應從「活躍列表」消失（後端 `active` 置為 0）。

### ✅ 顧客管理 Checkpoints
- [ ] **唯一性**：嘗試創建相同 Email 的顧客應妥善處理（或返回已存在）。
- [ ] **搜索功能**：在搜索框輸入關鍵詞，列表應實現前端模糊匹配。

### ✅ 訂閱與訂單 Checkpoints
- [ ] **狀態標籤**：`active` 訂閱應顯示黑色背景標籤；`canceled` 應顯示灰色。
- [ ] **金額格式**：金額應顯示為 `NT$ X,XXX` 格式。

### ✅ 統計數據 Checkpoints
- [ ] **MRR 計算**：新增一個 1000 元/月的產品訂閱後，MRR 應增加 1000。
- [ ] **待處理事項**：有未支付發票時，「待付款發票」計數應精確反映數據庫 `open` 狀態的數值。

## 3. 開發者驗證命令 (Dev Commands)

*   **後端狀態檢查**：`curl http://localhost:8080/api/v1/stats/overview`
*   **數據庫直接查詢**：`sqlite3 backend/seeder_local.db "SELECT * FROM core_product;"`
